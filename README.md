# SentinelAI Fusion

AI-powered phishing detection and SOC-assist platform built for rapid threat triage.

SentinelAI Fusion combines a FastAPI backend and a Next.js frontend to analyze emails, URLs, headers, and QR codes using a multi-layer detection pipeline (NLP, URL intelligence, visual analysis, header validation, and IOC correlation).

## Overview
SentinelAI Fusion is designed to support SOC analysts with:
- Real-time phishing analysis and risk scoring
- Explainable model breakdown and recommended actions
- Threat intelligence correlation with campaign/actor context
- Analyst chatbot support (RAG + operational context)
- Dashboarding, campaign views, and historical tracking

Default local ports:
- Backend: 8001
- Frontend: 3002

## Key Features
- 5-layer phishing detection pipeline:
  - NLP intent and manipulation tactic analysis
  - URL reputation and structural risk scoring
  - Visual similarity sandboxing (optional)
  - Email header authentication/anomaly detection
  - Threat intelligence and IOC feed enrichment
- Fusion scoring + verdicts: SAFE, SUSPICIOUS, PHISHING, CRITICAL
- Quishing detection (QR decode + URL analysis)
- Bulk analysis workflow
- Gmail integration (OAuth-based, optional)
- WebSocket streaming of analysis progress
- SentinelChat assistant for triage and response guidance
- Dashboard APIs for metrics, feed, timeline, campaigns

## System Architecture

```text
Browser (Next.js)
  -> REST + WebSocket
FastAPI Backend
  -> Multi-layer Detection Pipeline
  -> Fusion Engine
  -> Intelligence Correlation
  -> Optional External Services (OpenRouter, Apify, Replicate, Gmail, HIBP)
```

Runtime components:
- Backend service in `backend/`
- Frontend app in `sentAi/`

## Environment Setup
Create a `.env` file in `backend/`:

```env
# Core LLM / AI
OPENROUTER_API_KEY=
REPLICATE_API_TOKEN=
APIFY_API_TOKEN=

# Server
FRONTEND_URL=http://localhost:3002
PORT=8001

# Gmail OAuth
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_REDIRECT_URI=http://localhost:8001/api/v1/gmail/callback

# Dark web / breach intelligence
HIBP_API_KEY=
```

Create a `.env.local` file in `sentAi/`:

```env
NEXT_PUBLIC_API_URL=http://localhost:8001
NEXT_PUBLIC_WS_URL=ws://localhost:8001
```

## Running Locally

### 1) Start Backend

```bash
cd backend
python -m venv .venv
# Windows PowerShell:
.\.venv\Scripts\Activate.ps1
# macOS/Linux:
# source .venv/bin/activate

pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

Backend docs:
- Swagger UI: http://localhost:8001/docs
- ReDoc: http://localhost:8001/redoc
- Health: http://localhost:8001/health

### 2) Start Frontend

```bash
cd sentAi
npm install
npm run dev
```

Frontend URL:
- http://localhost:3002

## Deployment Notes
Minimum deployment shape:
- Deploy backend as Python ASGI service
- Deploy frontend as Next.js app
- Set environment variables for both services
- Allow CORS for frontend host
- Persist `backend/data/` only if historical cache retention is required

For production hardening:
- Place API behind TLS and a reverse proxy
- Add authn/authz on sensitive endpoints
- Restrict CORS to exact domains
- Add rate limits and request size controls
- Centralize logs and alerting

## Troubleshooting

Backend fails to start:
- Ensure venv is active
- Reinstall dependencies from `backend/requirements.txt`
- Check Python version is 3.11+

Frontend cannot reach backend:
- Confirm backend is running on port 8001
- Check `NEXT_PUBLIC_API_URL`
- Check browser/network tab for rewrite/proxy errors

Visual analysis unavailable:
- Ensure both `APIFY_API_TOKEN` and `REPLICATE_API_TOKEN` are set

Gmail endpoints disabled:
- Set `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`
- Confirm redirect URI matches your OAuth app settings

Slow first request:
- Expected due to startup warmup (knowledge graph, RAG, model loading)

## Security and Privacy Notes
- Treat all inbound content as untrusted
- Redact or tokenize PII before long-term storage when possible
- Avoid storing raw credentials or tokens in logs
- Use separate API keys for dev/staging/prod environments
